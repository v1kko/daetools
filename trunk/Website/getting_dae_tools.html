Getting DAE Tools

Introduction

pyDAE and cDAE modules come in a single package installed in pyDAE folder within site-packages (or dist-packages) folder under python (tipically /usr/lib/python26/Lib or C:\Python2.6\Lib). The structure of the folders is the following: 

daetools
  + cDAE
     + include
     + lib
  + daePlotter
  + daeSimulator
  + docs
     + api_ref
  + examples
  + pyAmdACML
  + pyDAE
  + pyIntelMKL
  + pyIntelPardiso
  + pyLapack
  + pyTrilinosAmesos

Requirements

Mandatory packages:

  • Python (v2.6.x): www.python.org
  • Boost libraries (v1.35+; GNU/Linux only): www.boost.org
  • Numpy (v1.3+): numpy.scipy.org
  • Matplotlib (v1.0+): matplotlib.sourceforge.net
  • pyQt4 (v4.0+): www.riverbankcomputing.co.uk/software/pyqt

Optional packages (3rd party linear solvers):

  • SuperLU (v3+; GNU/Linux only)
  • Umfpack (v5.4+; GNU/Linux only)
  • Lapack (GNU/Linux only)
  • Intel MKL (v10.2.5.035) - proprietary
  • AMD ACML (v4.4.0) - proprietary

For more information on how to install packages please refer to the documentation for the specific library.

Getting the packages

First download the appropriate installer for your operating system (GNU/Linux, Windows) and architecture (i386, amd64, arm):

  • .deb for Debian GNU/Linux and derivatives
  • .rpm for Red Hat GNU/Linux and derivatives
  • .exe for Windows

Installation

Debian GNU/Linux (and derivatives)

First install the mandatory packages. To do so you can use the Synaptic Package Manager or type the following command: 

$ sudo apt-get install libboost-all-dev python-qt4 python-numpy python-matplotlib

Optionally you may install some additional free linear solvers:

$ sudo apt-get install libsuitesparse-dev libsuperlu3 liblapack3gf

Install DAE Tools by using GDebi package installer or by typing the following shell commands: 

$ cd Directory_where_you_downloaded_the_package
$ sudo dpkg -i daetools_x.x-x_amd64_debian-6.deb


Red Hat GNU/Linux (and derivatives)

First install themandatory packages. To do so can use the Package Manager or type the following command: 

$ sudo yum install boost-devel PyQt4 numpy python-matplotlib

Optionally you may install some additional free linear solvers:

$ sudo yum install suitesparse-static superlu lapack

Note: there is no package superlu in Fedora GNU/Linux. However, you can use packages for instance from openSUSE (for Fedora 14 use packages from openSuse 11.3) http://download.opensuse.org/distribution/11.3/repo/oss/suse. Look for your  architecture and find the package superlu. Anyway, superlu package is needed only during the compilation (pyTrilinosAmesos module is statically linked to superlu.a library).

Install DAE Tools by using Package Installer or by typing the following shell commands: 

$ cd Directory_where_you_downloaded_the_package
$ sudo rpm -i daetools_x.x-x_amd64_fedora-14.rpm


Windows

So far it is tested on Windows XP only. First install the necessary dependencies (except the boost library, which is pre-built and comes with DAE Tools) in the following order: python, numpy, matplotlib, pyqt4. As a starting point you can use the following links:

  • Python 2.6: http://www.python.org/download/releases/2.6.6
  • Numpy: http://sourceforge.net/projects/numpy/files/NumPy
  • Matplotlib: http://sourceforge.net/projects/matplotlib/files/matplotlib/matplotlib-1.0
  • PyQt4: http://www.riverbankcomputing.co.uk/software/pyqt/download

To do so follow the instructions on the corresponding web sites. You can also try Python(x,y). Then install DAE Tools by double clicking the file daetools_x.x-x_win32_winxp_python26.exe and follow the instructions.


Additional linear equation solvers

Optionally you can also install proprietary AMD ACML and Intel MKL libraries. Please follow the installation procedures in the documentation. pyAmdACML and pyIntelMKL/pyIntelPardiso modules are compiled against ACML 4.4.0 and MKL 10.2.5.035 respectively. Also have a look on the licensing conditions (these libraries are not free software).

In order to use AMD ACML and Intel MKL libraries you have to do some additional configuration:

  1. GNU/Linux: setup for a single user
    Copy this file to your home folder, edit it so that it reflects your installation and add the line
    . $HOME/acml_mkl_bashrc at the end of $HOME/.bashrc file
  2. GNU/Linux: setup for all users
    Subject to your machine architecture and library versions (here x86_64 GNU/Linux with ACML v4.4.0 and MKL v10.2.5.035), put the following lines in /etc/ld.so.conf and execute ldconfig:
    /opt/intel/mkl/10.2.5.035/lib/em64t
    /opt/acml4.4.0/gfortran64_mp/lib
  3. Windows XP
    If not already added, add the following line to your PATH environment variable (Control Panel -> Sytem):
    c:\AMD\acml4.4.0\ifort32_mp\lib;c:\Intel\MKL\10.2.5.035\ia32\bin\


Compiling from source

Compiling of DAE Tools core libraries and python modules

To compile the source you need the following:

  • Installed boost library headers and compiled boost_system, boost_thread and boost_python libraries
  • Installed python 2.6
  • Installed python numpy module
  • Optionally: Trilinos Amesos, SuperLU, Lapack, Amd ACML and Intel MKL libraries.

All DAE Tools modules are developed using the QtCreator cross-platform integrated development environment. If you install the above dependecies correctly there is no much tweaking needed.
Check out the source code from the Google Code subversion repository:

$ cd "folder where you want the source"
$ svn checkout http://dae-tools.googlecode.com/svn dae-tools-read-only

GNU/Linux

  • Install the necessary dependencies (boost-devel, numpy)
  • Install QtCreator

Debian GNU/Linux based:

$ sudo apt-get install qtcreator

Red Hat GNU/Linux based:

sudo yum install qtcreator
  • Check dae-tools/trunk/dae.pri file
    Here you should check the following variables:
      - PYTHONDIR
        Tipically /usr/lib/python2.6 or /usr/lib64/python2.6 depending whether you are running
        32 or 64 bit GNU/Linux.
        Check the python version - it can be 2.7 in Fedora 14, for instance.
          - PYTHON_SITE_PACKAGES_DIR
        Typically it is $${PYTHONDIR}\site-packages, but can be $${PYTHONDIR}\dist-packages
            in Debian GNU/Linux based distributions.
      - PYTHON_INCLUDE_DIR
        You might need to add /usr/include/python2.6/numpy for Debian Squeeze if the header
        _numpyconfig.h is missing: Bug#589592.
  • Choose the right specification file for your platform (usually it is done automatically by the IDE, but double-check it):
      - for 32 bit machines use -spec linux-g++
      - for 64 bit machines use -spec linux-g++-64
  • Build the project dae through the QtCreator IDE or by the following commands:
$ cd "dae-tools/trunk"
$ qmake-qt4 dae.pro -r -spec linux-g++-64 CONFIG+=release
$ make
  • As a result there should be the following static/dynamic libraries in the debug/release folders:
      - libCore.a, libSimulation.a, libDataReporters.a, libSolver.a
      - pyCore.so, pyActivity.so, pyDataReporting.so, pySolver.so

Windows

  • Install the necessary dependencies (python 2.6, numpy)
  • Install MS Visual Studio 2008 or newer (Express Edition also works)
  • Download and unpack the boost library source to dae-tools/trunk/boost folder and build the libraries (look on http://www.boost.org/doc/tools/build/doc/html/index.html for the help)
  • Download and install Qt Open Source libraries for windows http://qt.nokia.com/products (for MS VC++)
  • Check dae-tools/trunk/dae.pri file
    Here you should check the following variables:
      - PYTHONDIR
        Tipically c:\Python26.
          - BOOSTDIR, BOOSTLIBPATH
        If you installed boost library to dae-tools/trunk/boost everything is fine;
        otherwise you have to point to the folder with the boost installation.
  • Build the project dae through the QtCreator IDE or by the following commands:
$ cd "to dae-tools\trunk"
$ qmake.exe dae.pro -spec win32-msvc2008 -r CONFIG+=release
$ make
  • As a result there should be the following static/dynamic libraries in the debug/release folders:
      - Core.lib, Simulation.lib, DataReporters.lib, Solver.lib
      - pyCore.pyd, pyActivity.pyd, pyDataReporting.pyd, pySolver.pyd


Note: You can also create Visual Studio projects. First edit dae.pri file as described above, open the Qt Command Prompt and use the following commands:

$ cd "to dae-tools\trunk"
$ qmake -tp vc -r dae.pro

Now you can open created project in the MS Visual C++ IDE.
